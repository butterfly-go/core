# Butterfly Core — Architecture

Internal architecture documentation for framework contributors and maintainers.

## Dependency Injection with Google Wire

The framework uses [Google Wire](https://github.com/google/wire) for compile-time dependency injection, replacing the previous function-chain initialization system with a type-safe, compile-time verified dependency graph.

### Package Layout

```
app/wire.go                  → Wire injector definition (build tag: wireinject)
app/wire_gen.go              → Generated initialization code (DO NOT EDIT)
app/app.go                   → Public API (`Config`, `Application`, `New`)

internal/bootstrap/bootstrap.go → Internal bootstrap orchestration and dependencies

internal/config/config.go    → ProvideConfig(), ProvideCoreConfig() providers
internal/config/registry.go  → SetConfig() / GetConfig() for process-global config access
internal/config/consul.go    → ConsulConfig implementation

internal/store/redis.go      → ProvideRedisClients() provider
internal/store/mongo.go      → ProvideMongoClients() provider
internal/store/sqldb.go      → ProvideSQLDBClients() provider
internal/store/s3.go         → ProvideS3Store() provider
internal/store/types.go      → Named types (RedisClients, MongoClients, etc.)
internal/store/registry.go   → Set/Get functions for internal state

store/redis/                 → Public API: GetClient(name) only
store/mongo/                 → Public API: GetClient(name) only
store/sqldb/                 → Public API: GetDB(name) only
store/s3/                    → Public API: GetClient(name), GetBucket(name) only
config/                      → Public API: Get(ctx, key) only
```

### Dependency Graph

Wire resolves dependencies by type at compile time:

```
mod.ConfigKey
    → ProvideConfig()        → config.Config (File or Consul backend)
    → ProvideCoreConfig()    → *mod.CoreConfig (parsed YAML)
        → ProvideRedisClients()  → store.RedisClients + cleanup
        → ProvideMongoClients()  → store.MongoClients + cleanup
        → ProvideSQLDBClients()  → store.SQLDBClients + cleanup
        → ProvideS3Store()       → *store.S3Store
```

Wire automatically chains cleanup functions and handles error rollback — if MongoDB initialization fails after Redis succeeds, Redis cleanup runs automatically.

### Startup Flow on `app.Run()`

```
1. Wire DI Phase
   initDeps(ConfigKey) → Dependencies + cleanup func

2. Internal bootstrap phase
   bootstrap.Init(service, configKey, appConfig, deps)
     → runtime.Init(service, configKey)
     → config.LoadAppConfig(...)   → yaml.Unmarshal → user AppConfig struct
     → log.Init()                  → slog.SetDefault()
     → metric.Init()               → otel.SetMeterProvider() + serve :2223
     → tracing.Init()              → otel.SetTracerProvider()
     → config.SetConfig(deps.Config)
     → store.SetRedisClients(deps.Redis)
     → store.SetMongoClients(deps.Mongo)
     → store.SetSQLDBClients(deps.SQLDB)
     → store.SetS3Store(deps.S3)

3. User InitFunc[] execution

4. Start servers (HTTP :8080, gRPC :9090)

5. Deferred shutdown path
   App.Close()
     → user TeardownFunc[] in reverse order
     → Wire cleanup func() for injected resources
```

### Key Files Reference

| File | Purpose |
|------|---------|
| `app/wire.go` | Injector definition — lists all providers in `wire.Build()` |
| `app/wire_gen.go` | Auto-generated by `wire` CLI — the actual initialization code |
| `app/app.go` | Public API surface; concrete app implementation stays package-private |
| `internal/bootstrap/bootstrap.go` | Internal startup orchestration for runtime/config/logging/registry setup |
| `internal/store/types.go` | Named types for Wire disambiguation (`RedisClients`, etc.) |
| `mod/types.go` | `ConfigKey` type for Wire |

### Registry Pattern

Public packages (`store/redis`, `config`, etc.) expose only `Get*()` functions. The `Set*()` functions live in `internal/` and are only called during the internal bootstrap phase:

```
app.Run()
    ↓ bootstrap.Init(...)
internal/store/registry.go
internal/config/registry.go
    ↑ calls internal Get*()
store/redis.GetClient()      ← public API
store/mongo.GetClient()      ← public API
config.Get()                 ← public API
```

### Regenerating Wire Code

After changing any provider signature or the injector definition:

```bash
go install github.com/google/wire/cmd/wire@latest
wire ./app/...
```

### Adding a New Provider

1. Create `ProvideXxx(deps...) (Type, func(), error)` in the relevant `internal/` package
2. Add the new type to `Dependencies` struct in `internal/bootstrap/bootstrap.go`
3. Add the provider to `wire.Build()` in `app/wire.go`
4. Run `wire ./app/...` to regenerate
5. Use the new dependency in `internal/bootstrap` or `app.Run()`, depending on whether it is internal bootstrap state or runtime orchestration
6. If public access is needed:
   - Add `SetXxx()` / `GetXxx()` to `internal/.../registry.go`
   - Call `SetXxx()` during `bootstrap.Init(...)`
   - Create a public package with `GetXxx()` that delegates to internal
